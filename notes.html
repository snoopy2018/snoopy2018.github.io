URL Confusion

Use relative URLs.

For CSS style sheets, the base URL is that of the style sheet, not that of the source document. For example, suppose the following rule:

body { background: url("yellow") }

is located in a style sheet designated by the URI:

http://www.example.org/style/basic.css

The background of the source document's BODY will be tiled with whatever image is described by the resource designated by the URI

http://www.example.org/style/yellow

(W3C Recommendation. Syntax and basic data types. Accessed November 15, 2019)

  • Icons you can only dream of: Icon Reference
  • Media query max- or min-. @media only screen and (max-width: 600px) { /* If the browser window is 'max-width' or smaller then... */ .text-block { font-size: 1.2em; } } @media only screen and (min-width: 600px) { /* If the browser window is 'min-width' or greater then... */ .text-block { font-size: 1.0em; } }
  • Use percentage for column width, but px for viewport size in media query. /* Extra small devices (phones, 600px and down) */ @media only screen and (max-width: 600px) { /* If the browser window is 'max-width' or smaller then... */ [class*="col-"] { width: 100%; } }
  • Typical breakpoints for various of devices. /* Small devices (portrait tablets and large phones, 600px and up) */ @media only screen and (min-width: 600px) {...} /* Medium devices (landscape tablets, 768px and up) */ @media only screen and (min-width: 768px) {...} /* Large devices (laptops/desktops, 992px and up) */ @media only screen and (min-width: 992px) {...} /* Extra large devices (large laptops and desktops, 1200px and up) */ @media only screen and (min-width: 1200px) {...}
  • Hide elements. /* If the screen size is 600px wide or less, hide the element */ @media only screen and (max-width: 600px) { div.example { display: none; } }
  • Use the max-width property. Image scales down as it has to, but up only to its max width. Same for videos. img, video { max-width: 100%; height: auto; }
  • "You can use the media query min-device-width, instead of min-width, which checks the device width, instead of the browser width. Then the image will not change when you resize the browser window:'' (w3schools, RWD Images) /* For devices smaller than 400px: */ body { background-image: url('img_smallflower.jpg'); } /* For devices 400px and larger: */ @media only screen and (min-device-width: 400px) { body { background-image: url('img_flowers.jpg'); } }
  • "The < picture > element works similar to the < video > and < audio > elements. You set up different sources, and the first source that fits the preferences is the one being used:" (w3schools, RWD Images) <picture> <source srcset="img_smallflower.jpg" media="(max-width: 400px)"> <source srcset="img_flowers.jpg"> <img src="img_flowers.jpg" alt="Flowers"> <!-- fallback --> </picture>
  • "The value of the max-width property overrides width." (CSS Height/Width); visit also CSS Max-width. The code below let content of div.ex2 shrink when browser window gets smaller than 500px wide, and does not scale anymore at 400px or less. div.ex2 { max-width:500px; min-width:400px; margin: auto; border: 3px solid #73AD21; }
  • "The solution that works in all browsers, is to set a default font-size in percent for the < body > element:" (CSS Fonts) body { font-size: 100%; } h1 { font-size: 2.5em; } p { font-size: 0.875em; }
